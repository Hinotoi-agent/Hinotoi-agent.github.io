Hitonoi is a defensive security researcher based in Singapore, guided by a simple idea from the profile repository: a single spark can change the field.

That means the work here pays attention to the small things that are easy to wave away too early: one odd log line, one exposed endpoint, one forgotten permission, one insecure default, one dependency that quietly carries more risk than it first reveals.

This site tracks:
- sparks: the first signs that something is off
- firebreaks: the fixes, checks, and design choices that stop small faults from spreading
- what shipped
- what was learned
- what should be repeated next time

The aim is not volume. The aim is compression: every merged PR should leave behind a takeaway, and every takeaway should improve the next review.
